rails2.3下用bundler

bundler是解决rails项目依赖性的很好的方式，在rails3下默认加入了bundler的支持，rails2.3想用bundler的话，需要增加和修改几个文件。

具体修改如下：

1、在config/boot.rb中的Rails.boot!行下面添加如下代码

class Rails::Boot
  def run
    load_initializer

    Rails::Initializer.class_eval do
      def load_gems
        @bundler_loaded ||= Bundler.require :default, Rails.env
      end
    end

    Rails::Initializer.run(:set_load_path)
  end
end

2、创建config/preinitializer.rb，添加如下代码

 

begin
  require "rubygems"
  require "bundler"
rescue LoadError
  raise "Could not load the bundler gem. Install it with `gem install bundler`."
end

if Gem::Version.new(Bundler::VERSION) <= Gem::Version.new("0.9.24")
  raise RuntimeError, "Your bundler version is too old for Rails 2.3." +
   "Run `gem install bundler` to upgrade."
end

begin
  # Set up load paths for all bundled gems
  ENV["BUNDLE_GEMFILE"] = File.expand_path("../../Gemfile", __FILE__)
  Bundler.setup
rescue Bundler::GemNotFound
  raise RuntimeError, "Bundler couldn't find some gems." +
    "Did you run `bundle install`?"
end

 

3、把原先项目中所有的config.gem全部删除，依赖的gem加入到Gemfile中。

ubuntu下gem install mysql报错

昨天在ubuntu下gem install  mysql时报错，

ERROR: Error installing mysql2: 

ERROR: Failed to build gem native extension

出现上面错误的原因，是本地缺少mysql的ruby类库和开发库

解决方案：

1、sudo apt-get install libmysql-ruby libmysqlclient-dev

2、sudo gem install mysql — –with-mysql-lib=/usr/lib/mysql/lib

Ruby的include和extend

在ruby中基本上有三种引入module的方式

一、在类定义中引入module后，module中的方法成为类的实例方法。

        在类定义中用include引入module。

       例如：

  1. module Base
  2.      def    test
  3.            p "This is a instance method!"
  4.      end
  5. end
  6.  
  7. class Car
  8.      include base
  9. end
  10. Car.new.test       =>         "This is a instance method!"

 

二、在类定义中引入module后，module中的方法成为类的类方法。

        在类定义中用extend引入module。

        例如：

  1. module Base
  2.      def    test
  3.            p "This is a instance method!"
  4.      end
  5. end
  6.  
  7. class Car
  8.      extend base
  9. end
  10. Car.test       =>         "This is a class method!"

 

三、在类定义中引入module后，module中的方法即有成为类的实例方法，又有成为类的类方法。

       这需要在类定义中引入module时，用include。

       例如：

 

  1. module Base
  2.      def    test
  3.            p "This is a instance method!"
  4.      end
  5.      def  self.included(base)
  6.           def base.call
  7.                p "This is a class method---call"
  8.           end
  9.           base.extend(ClassMethods)
  10.      end
  11.      module   ClassMethods
  12.           def hello
  13.                p "This is a class method----hello"
  14.           end
  15.      end
  16. end
  17.  
  18. class Car
  19.      include base
  20. end
  21.  
  22. Car.new.test       =>         "This is a instance method!"
  23. Car.call                =>         "This is a class method---call"
  24. Car.hello             =>         "This is a class  method---hello"

Rails plugin的介绍、安装和生成

1、什么是rails plugin? 

     rails plugin是对rails的扩展和修改
 
2、安装plugin
 
     把相应的plugin下载到rails apps的vendor/plugins目录下即可或者在rails apps根目录下执行如下命令：
        
  1. rails plugin install plugin_url(plugin的地址)
     
 
3、生成plugin
     下面做一个简单的plugin，实现的功能是在view中，调用say_hello(“kimi”)方法，返回Hello kimi字符串。
     即：<%= say_hello "kimi"%>  =>  "Hello kimi"
     在rails apps根目录下执行如下命令：
        
  1. rails generate plugin hello
     
     该命令会在vendor/plugin/目录下生成hello的目录，结构如下：
              create  vendor/plugins/hello
      create  vendor/plugins/hello/MIT-LICENSE
      create  vendor/plugins/hello/README
      create  vendor/plugins/hello/Rakefile
      create  vendor/plugins/hello/init.rb
      create  vendor/plugins/hello/install.rb
      create  vendor/plugins/hello/uninstall.rb
      create  vendor/plugins/hello/lib
      create  vendor/plugins/hello/lib/hello.rb
      invoke  test_unit
      inside    vendor/plugins/hello
      create      test
      create      test/hello_test.rb
       create      test/test_helper.rb

     其中在lib目录下存放逻辑代码，在lib/hello.rb文件中编写我们的代码。

  1.  module Hello
        def say_hello name
               "Hello #{name}!"
        end
     end

 

4、配置plugin

     经过以上几步，我们已经完成了功能实现代码，但是现在ActionView还不知道say_hello方法，所以不能在view中调用say_hello，
     如果要把say_hello加入到ActionView，需要在init.rb文件中增加如下代码：
  1. require 'hello'
    ActionView::Base.send :include,Hello
     或者
  1. require 'hello'
    ActionView::Base.class_eval do 
          include Hello
    end 
         
     rails会在应用启动的时候，加载vendor/plugins目录下的所有plugin，执行plugin的init.rb
     以上代码是把模块Hello动态include到ActionView中去，这样我们就可以在view页面中使用say_hello方法。

SVN的安装和配置

 

subversion在线教程：http://svndoc.iusesvn.com/svnbook/1.4/ 

一、准备工作
1、svn的安装步骤：http://wiki.centos.org/HowTos/Subversion 
2、etc/httpd/conf.d/subversion.conf 内容 
<Location /repos> 
        DAV svn 
        SVNPath /var/www/svn/ehaohai 
        AuthType Basic 
        AuthName "Subversion repos" 
        AuthUserFile /etc/svn-auth-conf 
        AuthzSVNAccessFile /etc/svn-acl-conf 
        Require valid-user 
</Location> 

二、备份 
http://wiki.archlinux.org/index.php/Subversion_backup_and_restore 

三、增加用户 
在linux命令行下输入vi etc/svn-acl-conf,显示如下： 
[groups] 
ehaohai = gaosy,wangcx,wangxl 
student  = zhangy 
[/] 
@ehaohai = rw 
@student = r 

如上所示，分为两个组ehaohai和student，其中ehaohai具有读写的权限，student只具有读的权限，要把某个用户添加到某个组，只需在后面 
加上用户名，同时用“，”隔开。 
以上为添加用户，为用户添加密码，执行如下命令 
htpasswd -cm /etc/svn-auth-conf  yourusername 
执行如上命令后，会出现提示，要求输入新密码，输入后则会把svn-auth-conf中的原内容删除（慎用），用添加的用户名密码替换原内容，如果不替换原内容基础上添加用户名和密码，则执行如下命令 
htpasswd -m /etc/svn-auth-conf otherusername
